# 网页版 AI 视频生成平台 MVP

一个仅供个人创作者本机自用试玩的网页应用（在浏览器里打开使用，不打包成任何桌面客户端 / 移动端 App）。用户在一个输入框中描述想要的视频内容，可选地附带一张首帧参考图，提交后由后端生成一个视频；成功生成的视频汇集到一个时间倒序的历史菜单中，可在本浏览器内反复查看、下载、重命名、删除。

## 使用者与场景

- **使用者**：单个个人创作者，自用试玩 / 体验向。
- **场景**：本人在桌面端浏览器中打开网页，立刻可用，无需注册登录，所有数据仅留在本浏览器内。

## 做什么

### 输入与提交
- 提供一个 prompt 文本输入框，用户输入对要生成视频内容的自然语言描述。
- 输入区域支持可选地上传 **一张** 首帧参考图片，已上传后允许移除再换一张。
- 提供"提交生成"按钮：
  - 不传图 → 走文生视频路径。
  - 传了图 → 走图生视频路径，以该图作为视频首帧。

### 生成过程
- 同一时刻只允许 **一个** 进行中的生成任务；任务进行期间，输入框与提交按钮被锁定/禁用。
- 生成期间在 UI 上显示"生成中"等待状态（具体形式由 design 决定）。
- 生成中的任务 **不可取消**，必须等到成功或失败终态。

### 任务终态
- **成功**：
  - 主区域展示并可播放刚生成的视频；
  - 同时把该次生成作为一条历史记录追加进历史菜单；
  - 输入框与提交按钮恢复可用。
- **失败**：
  - 在 UI 上显示失败提示（错误信息可见即可，具体样式由 design 决定）；
  - 输入框与提交按钮恢复可用；
  - **不在历史菜单留任何记录**。

### 历史菜单
- 列出所有 **成功** 的生成记录，按时间倒序（最新在最上方）。
- 每条记录至少携带：视频文件、当时使用的 prompt、当时使用的首帧图（如有）、生成时间、用户自定义标题（如有）。
- 每条记录支持以下操作：
  - **预览/播放**：在主区域加载并播放该次视频，同时能看到当时的 prompt 与首帧图（如有）。
  - **下载视频到本地**：把该次生成的视频文件保存到用户本地。
  - **删除**：从历史中移除该条记录。
  - **重命名 / 加标题**：为该条记录起一个自定义标题，覆盖默认标识（如默认的 prompt 截断或时间戳）。
- 历史 **持久化在本浏览器**：刷新页面、关闭再次打开浏览器仍可见。

## 不做什么

### 交互形态
- 不做多轮对话上下文：每次提交都是独立任务，AI 不会记住前几次说过的话。
- 不做基于已有视频的迭代 / 二次生成（即不支持"视频 → 视频"形态）。
- 不做"对历史记录追加新指令继续修改"。

### 输入相关
- 不做尾帧参考图、不做多张参考图、不做参考视频。
- 不做对 prompt 的 AI 改写 / 模板 / 灵感推荐。

### 参数与可调项
- 不暴露任何视频生成参数控件（时长、宽高比、分辨率、帧率、风格、模型选择等全部使用后端默认值）。

### 任务管理
- 不做并发：提交一个就必须等完，不允许提前提交下一个排队。
- 不做任务取消按钮。
- 不做失败任务历史：失败仅提示，不在历史菜单中留档；不提供"重试"按钮。

### 历史菜单
- 不做搜索 / 关键词过滤。
- 不做按日期 / 标签 / 收藏等任意筛选。
- 不做分类 / 文件夹 / 收藏夹。

### 账号与数据
- 不做账号体系：无注册、无登录、无个人资料。
- 不做跨设备 / 跨浏览器同步：清浏览器缓存、换设备、换浏览器都会丢失历史。
- 不做云端历史备份。

### 分享与外发
- 不做对外分享链接。
- 不做"一键发到 X 平台"等社交集成。
- 用户想发给别人 → 自行下载到本地后手动发送。

### 适配与外观
- 不打包为桌面客户端（Electron / Tauri 等）或移动端 App，仅作为网页在浏览器中使用。
- 不做手机 / 平板浏览器适配，仅保证桌面端浏览器体验。
- 不做暗色模式 / 多语言（MVP 默认一套外观即可）。

## 验收

- [ ] 打开网页 **不需要任何登录或注册**，立刻进入可用状态。
- [ ] 用户能在输入框中输入 prompt 文本。
- [ ] 用户能上传 **一张** 首帧参考图，且能在提交前移除并重新上传。
- [ ] 用户 **不上传图** 也能成功提交并生成（文生视频路径）。
- [ ] 用户 **上传了图** 时，生成的视频以该图为首帧（图生视频路径）。
- [ ] 提交后，输入框与提交按钮处于锁定/禁用态，UI 上明确显示"生成中"。
- [ ] 同时只能存在一个进行中的任务：进行中无法再次提交。
- [ ] 进行中无任何可点击的"取消"入口。
- [ ] 任务 **成功** 后：
  - [ ] 主区域可播放刚生成的视频；
  - [ ] 历史菜单顶部新增一条该次生成的记录；
  - [ ] 输入框与提交按钮恢复可用。
- [ ] 任务 **失败** 后：
  - [ ] 用户能在 UI 上看到失败提示；
  - [ ] 历史菜单中 **不出现** 该次失败记录；
  - [ ] 输入框与提交按钮恢复可用。
- [ ] 历史菜单按生成时间 **倒序** 排列（最新在最上方）。
- [ ] 点击历史中任意一条，主区域能加载并播放该次视频，且能看到当时的 prompt；如果当时上传过首帧图，也能看到该首帧图。
- [ ] 历史中每条记录能 **下载** 视频文件到用户本地。
- [ ] 历史中每条记录能 **删除**，删除后立即从列表移除且不可恢复。
- [ ] 历史中每条记录能 **重命名 / 加自定义标题**，刷新页面后自定义标题仍然保留。
- [ ] 在桌面浏览器中刷新页面后，历史菜单中的所有记录（含视频本身、prompt、首帧图、自定义标题）依旧可见。
- [ ] 关闭浏览器后再次打开网页，历史菜单中的所有记录仍然可见。
