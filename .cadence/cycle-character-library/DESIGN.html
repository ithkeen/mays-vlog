关键流程

打开抽屉

1. 用户点 header 右上 Characters 按钮 → 父组件设 openDrawer = 'characters'
2. 若先前是 'history'，HistoryDrawer 同帧退出
3. CharacterDrawer 首次挂载 → listCharacters() → 渲染网格
4. 默认进入「列表态」

创建角色

1. 点「新建角色」占位卡 → 抽屉切到「创建表单态」（顶部 ← 返回 | 新建角色）
2. 选图片：<input accept="image/png,image/jpeg,image/webp">；提交前 MIME 二次校验；选中后 <img> 预览
3. 填 Name（必填）+ Instructions（可选）
4. 点「创建」 → 校验通过 → 写 IDB → 列表 prepend → 切回列表态；失败 → 表单内显示错误，不切态
5. 点「取消」/「返回」 → 切回列表态，丢弃已选图片与输入

查看 / 删除角色

点卡片头部 → 原地内联展开看完整 Instructions；多卡可并存展开
展开态卡内点「删除」 → 同位置变态为「确认删除 | 取消」
「确认删除」 → deleteCharacter(id) → 从列表移除 → revoke 该卡用过的 object URL
「取消」 → 回到普通展开态

关键决策

  • 复用 mays-vlog IDB 新增 store，不开新 DB：与 PROJECT.md idb 约定一致；多 store 是 idb 标准用法；未来导出/迁移更方便。
  • 图片存 Blob 不存 base64：IDB 原生支持，无 ~33% 膨胀；渲染走 URL.createObjectURL。
  • CharacterDrawer 与 HistoryDrawer 同侧互斥：用统一的 openDrawer 单值状态实现，避免两个独立 boolean 互相不知道。
  • 创建表单走「抽屉两态切换」：贴合「不弹 modal、不跳页」；表单空间充裕，逻辑简单。
  • 删除二次确认走「卡内内联变态」：同样贴合「不弹 modal」原则。
  • Name 唯一性 trim + toLowerCase：用户视角下「Alice」「 alice」是同一个角色；存储保留原大小写用于展示。
  • 入口按钮放 header 右上与 History 并排：所见即所得；与现有抽屉触发模式一致。
  • 不接入生成流程：留给下一轮 cycle。
