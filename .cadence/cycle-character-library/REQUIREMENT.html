Character 库

在 mays-vlog 里加一个本浏览器持久化的角色资料库，把你想反复用的角色（人物 / 动物 / IP）存起来，作为后续长视频生产与跨场景角色一致性管理的数据底座。本轮 cycle 只交付角色库的「增 / 查 / 删」，不接入现有的视频生成流程。

本轮交付一览

目标用户
单个个人创作者，本机浏览器自用（与 mays-vlog 现有口径一致）
核心价值
把"角色"沉淀为可命名、可复用的本地数据，为下一轮 cycle 接入"选角色生成"打地基
入口位置
主界面新增一个侧折抽屉，独立于现有历史抽屉
角色字段
参考图（必填，仅本地上传）+ Name（必填、唯一）+ Instructions（可选自由文本）
数据范围
仅本浏览器持久化（清缓存 / 换设备 / 换浏览器会丢，与现有历史一致）

范围

✅ 做什么

  • 主界面新增 Character 库侧折抽屉，与历史抽屉互不干扰
  • 抽屉内卡片网格展示全部角色（参考 Runway Characters 列表风格）
  • 网格首格固定为「新建角色」占位卡
  • 创建：上传一张本地图 + 必填 Name + 可选 Instructions
  • 查看：卡片原地内联展开看完整 Instructions（不弹 modal）
  • 删除：必须二次确认
  • 列表默认按创建时间倒序
  • 数据本浏览器持久化，刷新 / 重启浏览器不丢

❌ 不做什么

  • 不做 Voice、不做 Knowledge base（Runway 上有，本轮明确去掉）
  • 不做 Try examples 预设、不做 "Optimize image for quality" 选项
  • 不做编辑（改名 / 换图 / 改描述都不支持，要改只能删了重建）
  • 不做多参考图，一个角色仅一张主图
  • 不做"从历史视频抽帧"作为图源
  • 不做跨设备 / 跨浏览器同步
  • 不设角色数量上限
  • 不做搜索 / 标签 / 分组 / 收藏 / 置顶 / 手动排序
  • 不做角色卡片下方的身份/职业标签
  • 不接入生成流程，"选角色一起生成"留给下一轮 cycle

验收标准

  • 用户在主界面能找到并打开 Character 库侧折抽屉，且与现有历史抽屉互不影响
  • 用户能在抽屉内点击「新建角色」卡，进入创建流程
  • 用户能上传一张本地图、填 Name、可选填 Instructions，提交后角色立即出现在列表里
  • Name 未填时无法创建，且给出明确错误提示
  • Name 与已有角色重名时无法创建，且给出明确错误提示
  • 卡片默认显示图 + Name；点击能原地内联展开看到完整 Instructions，再次点击折叠
  • 展开状态的卡片里能触发删除，且必须二次确认才真正删除
  • 删除后角色立即从列表消失
  • 列表默认按创建时间倒序排列
  • 刷新页面、重启浏览器后角色与参考图均仍然存在
  • 清缓存 / 换浏览器 / 换设备后看不到先前角色（与现有历史一致，符合预期）
  • 本轮 cycle 不修改现有 prompt 生成流的任何 UI 与 API；现有生成功能行为不变
