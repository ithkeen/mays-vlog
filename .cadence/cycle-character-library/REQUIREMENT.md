# Character 库

mays-vlog 新增一个本浏览器持久化的角色（character）资料库，作为后续长视频生产与跨场景角色一致性管理的数据底座。本轮 cycle 只交付角色库的「增 / 查 / 删」三件套，不接入任何生成流程。

## 目标用户与使用场景
- 单个个人创作者，本机浏览器自用（沿用 PROJECT.md 已有口径）。
- 在创作前后，把可复用的角色（人物 / 动物 / IP 等）攒进库里，方便后续 cycle 把它们作为生成依据。

## 核心价值
- 把"角色"沉淀为可命名、可复用的本地数据；为下一轮 cycle 接入角色一致性生成做准备。

## 做什么
- 新增**主界面侧折抽屉**作为 Character 库入口，独立于现有 HistoryDrawer，不与历史抽屉混在一起。
- 抽屉内以**卡片网格**展示全部角色，整体风格参考 Runway Characters 列表页（图 + Name）。
- 网格首格固定为「新建角色」占位卡，点击进入创建表单。
- 创建角色：
  - 上传一张**本地图片**作为角色参考图（必填，仅本地上传）。
  - 填写 **Name**（必填、全局唯一，禁止与已有角色重名）。
  - 填写 **Instructions**（可选，自由文本，给角色加额外说明）。
  - 提交后角色立即出现在列表里。
- 查看角色：
  - 卡片默认显示「参考图 + Name」。
  - 点击卡片在列表内**原地内联展开**，显示完整 Instructions；再次点击折叠。**不弹 modal、不跳页**。
- 删除角色：
  - 入口在展开后的卡片内。
  - 点击删除**必须经过二次确认**才真正删除。
  - 删除后角色立即从列表消失。
- 数据**仅本浏览器持久化**：刷新页面、重启浏览器后角色依旧在；清缓存 / 换浏览器 / 换设备会丢，与现有历史口径一致。
- 列表默认按**创建时间倒序**排列。

## 不做什么
- ❌ 不做 Voice、不做 Knowledge base（Runway 上有，本轮明确去掉）。
- ❌ 不做 Try examples 预设角色、不做 "Optimize image for quality" 选项。
- ❌ 不做编辑：改名 / 换图 / 改描述都不支持，要改只能删了重建。
- ❌ 不做多参考图：一个角色仅一张主参考图。
- ❌ 不做"从历史视频里抽帧"作为角色图来源。
- ❌ 不做跨设备 / 跨浏览器同步。
- ❌ 不设角色数量上限。
- ❌ 不做搜索 / 标签 / 分组 / 收藏 / 置顶 / 手动拖拽排序。
- ❌ 不做角色卡片下方的身份/职业标签（Runway 示例里 Influencer / Music Creator 那种，本轮没这个字段）。
- ❌ **不接入生成流程**：本轮 cycle 不在 prompt 生成里"选角色一起生成"，这件事留给下一轮 cycle。

## 验收标准
- [ ] 用户在主界面能找到并打开 Character 库侧折抽屉，且与现有历史抽屉互不影响。
- [ ] 用户能在抽屉内点击「新建角色」卡，进入创建流程。
- [ ] 用户能上传一张本地图片、填 Name、可选填 Instructions，点击创建后角色立即出现在列表里。
- [ ] Name 未填时无法创建，且给出明确错误提示。
- [ ] Name 与已有角色重名时无法创建，且给出明确错误提示。
- [ ] 列表中的角色卡片显示参考图 + Name；点击卡片能**原地内联展开**看到完整 Instructions，再次点击折叠。
- [ ] 展开状态的卡片里能触发删除，且**必须二次确认**才会真正删除。
- [ ] 删除后角色立即从列表消失。
- [ ] 列表默认按创建时间倒序排列。
- [ ] 刷新页面、重启浏览器后角色与参考图均仍然存在。
- [ ] 清缓存 / 换浏览器 / 换设备后看不到先前角色（与现有历史一致，符合预期）。
- [ ] 本轮 cycle 不修改现有 prompt 生成流的任何 UI 与 API；现有生成功能行为不变。
