网页版 AI 视频生成平台 MVP

一个仅供个人创作者本机自用试玩的网页应用 · 在浏览器里打开使用，不打包成任何桌面或移动端客户端

打开网页就能用，无需登录。在输入框里写一段对视频内容的描述，可以选择上传一张首帧图，点击生成即可。一次只跑一个任务，成功后视频会出现在主区域，并自动归档到右侧历史菜单中，可随时回看、下载、重命名或删除。

使用者

单个个人创作者，自用试玩

核心场景

桌面端浏览器打开即用，本机自存

核心交互

单次 prompt 输入 + 可选首帧图

历史持久化

本浏览器内，清缓存即丢失

范围

✅ 做什么

  • 单个 prompt 输入框，提交一次生成一个视频
  • 可选上传 一张 首帧参考图（同时支持文生视频与图生视频）
  • 同时只允许一个进行中的任务，进行期间输入区锁定
  • 成功的视频在主区域可播放
  • 成功的生成自动进入历史菜单（按时间倒序）
  • 历史每条记录支持：预览播放 / 下载 / 删除 / 重命名
  • 历史持久化在本浏览器，刷新和重开浏览器仍在

❌ 不做什么

  • 不做多轮对话，不做"基于已有视频继续修改"
  • 不做尾帧图、不做多张参考图、不做参考视频
  • 不暴露任何参数控件（时长 / 宽高比 / 分辨率等全用默认值）
  • 不做并发任务、不做任务取消、不做失败重试按钮
  • 失败任务不入历史
  • 历史菜单不做搜索 / 筛选 / 分类 / 收藏
  • 不做账号、不做登录、不做跨设备同步
  • 不做对外分享链接、不做社交平台一键发布
  • 不打包为桌面客户端 / 移动端 App，纯浏览器网页
  • 不适配手机 / 平板浏览器，不做暗色模式 / 多语言

验收标准

  • 打开网页 无需任何登录或注册，立刻可用
  • 能在输入框中输入 prompt 文本
  • 能上传 一张 首帧参考图，且提交前可移除并重新上传
  • 不传图 也能成功提交并生成（文生视频）
  • 传了图 时，生成的视频以该图为首帧（图生视频）
  • 提交后输入框与提交按钮被锁定，UI 明确显示"生成中"
  • 进行中无法再次提交，且无可见的"取消"入口
  • 任务成功：主区域可播放刚生成的视频，历史顶部新增一条记录，输入区恢复可用
  • 任务失败：能看到失败提示，历史菜单中 不出现 该次记录，输入区恢复可用
  • 历史菜单按生成时间 倒序 排列
  • 点击历史任意一条：主区域能播放该次视频，并能看到当时的 prompt 与首帧图（如有）
  • 历史中每条记录能下载视频文件到本地
  • 历史中每条记录能删除，删除后立即从列表移除
  • 历史中每条记录能重命名 / 加自定义标题，刷新后仍保留
  • 刷新页面后，历史中的所有记录（视频、prompt、首帧图、标题）仍可见
  • 关闭浏览器再次打开网页后，历史中的所有记录仍可见
