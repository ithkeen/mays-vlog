前端布局重构：sidebar 多入口 + 主区整页切换

本次纯前端重构：把现有的"主区 + 抽屉"形态改为"常驻窄态 sidebar + 主区路由整页切换"。引入 react-router-dom v6，用一个 AppShell 承载 sidebar + main outlet；三个一级页面 Generate / History / Characters 全部 keep-mounted，以 CSS display 切换可见性 —— 这是为了让 Generate 页的任务进度与轮询 hook 不因切页而丢失，同时满足"内部组件逻辑不动"的需求边界。HistoryDrawer / CharacterDrawer / openDrawer 抽屉互斥状态机整体删除；两者的内部子组件（HistoryDetail / CharacterCard / CharacterCreateForm 等）平提到 pages/ 与 components/{character,history}/ 下复用。

关键决策

  • 选 react-router-dom v6 不选自建 history hook：嵌套路由 + :id 参数提取零成本；PROJECT.md "前端 HTTP 不引第三方" 的节制原则针对 fetch，不覆盖路由领域
  • 三页全部 keep-mounted，用 CSS display 切换：既满足"切回 Generate 进度不丢"验收，又不违反"内部组件逻辑不动"。不选 Context 外提任务状态的原因就是那会重写 SubmissionWorkspace
  • 首帧缩略图走 <video preload=metadata> + currentTime=0.1：不动后端、不动 IDB schema；用 canvas 存 Blob 要 DB_VERSION+1 与需求"只动 UI 骨架"冲突
  • play_url 并发限流 6：避免 burst 拉爆 UFile 预签接口；MVP 历史量级下可见卡等待时间可忽略
  • /characters/new 与 /characters 共用 CharactersPage：需求是"主区切换为创建表单态"而非独立页；共用避免 grid ↔ form 切换时 unmount/mount 抖动
  • Character 卡原位纵向抽高：minimal-refined 基调要求布局安静；满行展开带来的整屏重排视觉噪音大

留到执行时

  • Sidebar 图标选型（Generate / History / Characters 三个 icon 的具体形状与来源库）
  • Sidebar 宽度在 64–80 px 区间的具体取值、卡片 radius / gap / 渐变叠层高度与透明度、展开卡动画参数
  • 返回 History 列表的交互形态（面包屑 vs 返回按钮 vs 两者）
  • /characters/new 下 CharactersPage 的表单与 grid 的具体布局关系
