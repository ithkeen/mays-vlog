前端布局重构：sidebar 多入口 + 主区整页切换

cyclefrontend-redesign2026-05-13

当前 MVP 前端是 "主区 SubmissionWorkspace + grid 左列常驻 HistoryDrawer + fixed 浮层 CharacterDrawer" 的 主区 + 抽屉 形态。本次重构对齐 Runway 主页视觉骨架——左侧改为常驻、icon + label 竖向窄 sidebar，承载 3 个一级入口（Generate / History / Characters），主区随选择整页切换；HistoryDrawer 与 CharacterDrawer 两个抽屉容器整体废除。Character 与 History 列表的展示样式参考 Runway Characters 列表页的 grid 大卡（缩略图满铺 + 底部渐变 + 标题）。视觉契约保留（minimal-refined / 浅色 / 冷色系 / accent #1d4ed8 / IBM Plex Sans）。

本次只动 UI 骨架与样式，不动内部能力（Character 编辑、Character 接入生成流程仍是已知限制）。

范围

做什么

  • 左侧常驻窄态 sidebar（icon + 下方小字 label），3 个一级入口 Generate / History / Characters，选中态加深 + 左侧细竖线
  • 主区随选择整页切换；URL 反映当前页（/generate / /history / /history/{id} / /characters / /characters/new）；刷新后保留在当前页
  • 主区顶部 header 仅左侧放当前页标题；右侧留空
  • Generate 页包装现有 SubmissionWorkspace，内部组件逻辑不动
  • History 页 grid 卡（视频首帧满铺 + 底部渐变 + 标题 + 完成时间）；点卡进 /history/{id} 独立详情页
  • Characters 页 grid 卡（参考图满铺 + 底部渐变 + Name）；首位永远是 + 占位卡
  • 点 + 占位卡 → 主区切换为创建表单态（路由 /characters/new），表单字段保留现状（参考图 + Name + Instructions）
  • 点已有角色卡 → 卡内联展开（Name / Instructions / 大图），再点收起；删除走"卡内二次确认"
  • 保留现有视觉契约：accent #1d4ed8 / IBM Plex Sans / 浅色冷色
  • 移除 HistoryDrawer / CharacterDrawer 抽屉容器、openDrawer 状态机、主界面 header 上原抽屉按钮

不做什么

  • 不补 Character 编辑（改名 / 换图 / 改描述）
  • 不补 Character 接入生成流程（Generate 选角色一起生成）
  • 不做跨页可见的生成进度（无全局状态栏 / 无 sidebar badge）
  • 不改视觉契约（主色 / 字体 / 间距 / 字号 token 都不动）
  • 不做 sidebar 折叠态（永远窄态，无 240px 展开）
  • 不放 sidebar 底部账户 / 设置 / 帮助 / 反馈入口
  • 不放主区顶部 header 右侧任何按钮（credits / Dev Portal / 新建 / 清空 / 设置等都不放）
  • 不做 History 卡 hover 删除/下载等快捷操作；不做 List ↔ Grid toggle
  • 不做移动 / 平板适配；不做暗色模式；不做多语言
  • 不做键盘快捷键（Cmd+1/2/3 切 sidebar 等）
  • 不动后端（API / DB schema / 任务编排都不变）
  • 不改内部组件逻辑（PromptInput / VideoPlayer / ProgressPanel / 表单字段都保留）
  • 不引入社区流 / 外部参考流 / inspirations
必须代码层面移除： HistoryDrawer 容器（grid 左列常驻 + 列宽塌缩开关）、CharacterDrawer 容器（fixed 浮层 + 列表/创建两态外壳）、openDrawer: 'none' | 'history' | 'characters' 互斥状态机、主界面 header 上原抽屉触发按钮。

验收

  • 用户进入应用默认看到 Generate 页，URL 是 /generate
  • 左侧能看到一条永远窄态的 sidebar，3 个一级入口（Generate / History / Characters）以 icon + 下方 label 竖向排列
  • 点击任一 sidebar 入口，主区整页切换到对应内容，URL 同步变化
  • 任一页刷新浏览器后停留在当前页（如刷新 /history 仍回到 History 而不是 Generate）
  • History 页能看到所有已成功的视频作品 grid 卡片，每张卡是"视频首帧满铺背景 + 底部渐变叠层 + 标题 + 时间"样式
  • History 页点击任一卡片 → 进入 /history/{id} 独立详情页，能播放视频并查看 prompt / 元数据
  • History 详情页能通过面包屑或返回按钮回到 History 列表
  • Characters 页能看到 grid 卡片，首位是带 + icon 的占位卡，其余每张是"参考图满铺背景 + 底部渐变叠层 + Name"样式
  • 点占位卡 → 主区切换为创建表单态，URL 变 /characters/new，能填参考图 / Name / Instructions
  • 创建表单态点提交或取消后回到 Characters 列表态（URL 回 /characters）
  • 点任一 Character 卡能卡内联展开，看到 Name / Instructions / 参考图大图；再点收起
  • Character 卡上点删除有"卡内二次确认"交互（与现状一致）
  • Generate 页发起生成后切到 History / Characters 页，应用任何位置都看不到全局进度提示
  • Generate 页发起生成后切到其他页再切回 Generate，能继续看到当前任务的进度（任务未中断、进度未丢失）
  • 应用代码中已无 HistoryDrawer / CharacterDrawer 抽屉容器组件，也无 openDrawer 抽屉互斥状态机
  • 应用整体视觉契约保持：accent 仍 #1d4ed8 / 字体仍 IBM Plex Sans / 浅色冷色基调不变
  • 现有 "Character 编辑" 与 "Character 接入生成流程" 两项已知限制本轮不消除（继续作为已知限制存在）
